--- Chat 6: Page ID Resolution Error Fix ---

USER: Getting error: can not solve page id for the page.

ASSISTANT: Root cause: PowerShell 5.1's Invoke-WebRequest with -MaximumRedirection 0
doesn't reliably expose Location headers from 302 responses via exceptions.

Fix: Replaced Invoke-WebRequest in Resolve-PageId function with .NET
HttpWebRequest which properly captures Location headers:
- System.Net.HttpWebRequest with AllowAutoRedirect = $false
- Manual redirect following loop (up to 6 hops)
- Catches Location header from both normal responses and WebException responses
- Added fallback: extracts tinyurl identifier and calls
  /pages/tinyurl.action?urlIdentifier= directly

--- Chat 7: DataGridView "No Columns" Error Fix ---

USER: Exception calling Add with four arguments. No row can be added to a data
grid view control that does not have columns.

ASSISTANT: Root cause: dgv.Rows.Clear() was also stripping columns when
AutoGenerateColumns wasn't explicitly disabled.

2. After Rows.Clear(), check if columns were lost and re-add them if needed

--- Chat 8: AddRange Type Casting Error Fix ---

USER: Cannot convert argument "dataGridViewColumns" with value "System.Object[]"
for "AddRange" to type "System.Windows.Forms.DataGridViewColumn"

ASSISTANT: Root cause: PowerShell wraps @(...) arrays in another System.Object[]
which .NET's AddRange can't cast to DataGridViewColumn[].

Fix: Replaced all $dgv.Columns.AddRange(@($col1, $col2, ...)) calls with
individual [void]$dgv.Columns.Add($col) calls.

Two locations fixed:
- Line 214 (initial column setup)
- Line 424 (column re-add in scan click handler)

KEY DECISIONS & LESSONS LEARNED:

1. SECURITY: Never hardcode API tokens in files. Use environment variables.

2. CONFLUENCE API:
   - Short URLs (/wiki/x/...) redirect through tinyurl.action
   - Footer comments use v2 API: POST /wiki/api/v2/footer-comments
   - v1 comment API (/rest/api/content/{id}/child/comment) returns 405

3. POWERSHELL 5.1 QUIRKS:
   - Invoke-WebRequest doesn't reliably expose Location headers from 302s
     -> Use System.Net.HttpWebRequest instead
   - DataGridView.Rows.Clear() can strip columns
     -> Set AutoGenerateColumns = $false
   - AddRange(@(...)) wraps array in Object[] causing type cast failure
     -> Use individual .Add() calls instead

4. ENVIRONMENT VARIABLES:
   - Set persistent: [Environment]::SetEnvironmentVariable("name", "val", "User")
   - Set session: $env:NAME = "value"
